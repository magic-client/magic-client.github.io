@magic

magic is a module based generator for interactive clients, apis and static pages.

this is also called a jamstack, javascript, apis and markup.

less than 15kb of html + css + javascript boilerplate means @magic is minimal, and the module based approach means that it is also easily extendable and customizable.

by creating html, css and javascript files, @magic makes sure that your pages load as quickly as possible, and your app will stay usable without javascript.

@magic-libraries

below is a collection of the available @magic client libraries.

  • @magic-libraries/db

    key => value store using localstorage

    docs / demo

  • @magic-libraries/gql

    the @magic-libraries/gql module encodes template strings to graphql queries ready to be sent to a server.

    docs / demo

  • @magic-libraries/is

    the @magic-libraries/is module unifies the javascript type testing apis.

    docs / demo

  • @magic-libraries/json

    the @magic-libraries/json module parses and stringifies json. it also returns errors instead of throwing them.

    docs / demo

  • @magic-libraries/prevent-default

    the @magic-libraries/prevent-default module invokes event.preventDefault as a hyperapp effect.

    docs / demo

  • @magic-libraries/uri

    the @magic-libraries/uri module encodes objects to uri strings and decodes uri strings to objects.

    docs / demo
